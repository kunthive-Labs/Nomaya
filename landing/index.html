Generic evals miss the failures that regulators fine.

The high-stakes failures in regulated finance are subtle: leaking nonpublic personal information, omitting a required Truth-in-Lending disclosure, a UDAAP-deceptive statement about fees, acting on a transfer without authorization. A benchmark score says nothing about any of them.

Nomaya runs an agent through regulation-mapped scenarios in a sandbox and checks its behavior against named obligations. It turns "does the agent behave compliantly?" into a measurable, CI-gateable test instead of a hope.

regulation-mapped scenarios

Every scenario names its rule

YAML playbooks tie each conversation to specific US and EU obligations: GLBA, UDAAP, Reg Z, Reg E, FCRA, ECOA, SR 11-7, NYDFS 500, the EU AI Act, DORA, GDPR, and CAN-SPAM. A failure points at the obligation it broke.

provider-agnostic

Any lab, one interface

The agent under test and the judge can be OpenAI, Anthropic, Google, Mistral, Cohere, Groq, or local Ollama, all through one LiteLLM interface. Adding a lab is a config string, not code.

deterministic mock mode

Runs with no API keys at all

Deterministic compliant and naive agent scripts run the full suite offline with zero token spend, which makes it CI-friendly. Mock mode is a first-class path, not a stub.

declarative checks

Seven check types, stated up front

required_phrase, prohibited_phrase, regex, no_pii_leak, tool_call_expected, tool_call_forbidden, and llm_judge, backed by a built-in PII and NPI detector.

honest metrics

Recall, not just a pass rate

Violation detection rate is reported as recall, the false-positive rate comes from benign control scenarios, and every run carries a severity-weighted score alongside cost and latency.

pass@k reliability

Once is not the same as always

Each scenario is run k times, because a model that complies once may not comply consistently. The suite measures the difference.

The first run costs nothing.

Python 3.11 or newer. The default run uses deterministic mock mode, so it works offline before you decide which model to point it at.

nomaya - zsh
$ uv sync --locked && source .venv/bin/activate $ nomaya run # full suite in deterministic mock mode, no API key $ nomaya run --agent openai/gpt-4o-mini --judge openai/gpt-4o-mini $ docker compose up --build # API on :8000, dashboard on :3000

A measurement tool, not a legal opinion.

Nomaya tells you how an agent behaved against a written expectation of an obligation. That is a test result, and it should be read as one.

Read before relying on the mappings. The regulation summaries in Nomaya are paraphrased for orientation and are not legal advice. Mappings should be reviewed by qualified compliance staff before relying on them.
